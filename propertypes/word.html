The Veloca Typography Spec

This document demonstrates how Veloca renders Markdown. We believe that writing is essentially about typography. Every spacing, every border, and every font weight has been meticulously tuned to achieve the Premium Shadcn Aesthetic.

1. Headings & Typography

Headings use negative letter-spacing for a sharper, more compact appearance. Notice the visual hierarchy and rhythm between H1, H2, and H3.

2. Text Formatting

Standard text is colored in deep gray (#3f3f46) rather than pure black to reduce eye strain. You can use Bold for emphasis, Italics for nuance, or strikethrough for deleted text. You can also highlight text using the equal signs syntax.

Links are designed with a generous underline offset. Check out the Shadcn UI Documentation to see where we got our inspiration.

3. Blockquotes & Callouts

Blockquotes are used to extract important text or quote external sources. They use a sleek left border instead of clumsy background colors.

5. Code & Engineering

Inline code blocks use a very subtle background and border. For multi-line code, we use fenced blocks with language tags.

// Veloca Core Render Function
import { unified } from 'unified'
import remarkParse from 'remark-parse'

export async function renderMarkdown(content: string) {
  const result = await unified()
    .use(remarkParse)
    .process(content)
    
  return String(result)
}

6. Tables

Tables are notoriously hard to make beautiful. We use border-collapse, slightly darker header backgrounds, and zebra-striping on hover.
