# I18n 配置模型

本文档描述 Veloca 的前端国际化配置。当前目标是建立项目级 i18n 基础设施，并让 Settings 语言偏好真实驱动界面文案切换。

## 设计原则

- i18n 先以轻量内置资源实现，不引入额外运行时依赖，避免 Electron 桌面应用增加不必要的包体积和加载风险。
- 语言偏好继续复用 Settings > Appearance 中的 `language` 设置，并保存到本地 `app_settings`。
- `system` 语言会根据 `navigator.language` 解析为 `en` 或 `zh-CN`；未知语言默认回退到 `en`。
- 翻译资源集中在 `app/frontend/src/i18n.ts`，新增界面文案必须通过该模块的 `t()` 读取，避免继续散落硬编码 UI 文案。

## 当前语言

Veloca 当前支持：

- `en`：英文
- `zh-CN`：简体中文
- `system`：跟随系统，运行时解析为英文或简体中文

## 前端使用方式

`App.tsx` 会根据 `appearanceSettings.language` 创建翻译函数：

```tsx
const resolvedLanguage = useMemo(() => resolveLanguage(appearanceSettings.language), [appearanceSettings.language]);
const t = useMemo(() => createTranslator(resolvedLanguage), [resolvedLanguage]);
```

组件中使用稳定 key 获取文案：

```tsx
<span>{t('settings.appearance.language')}</span>
```

需要插值的文案使用参数对象：

```tsx
t('settings.typography.fontSizeHint', { size: defaultEditorFontSize });
```

## 当前接入范围

- Settings 侧边导航、Editor、Appearance、Typography、Shortcuts、AI Model、Remote、About Veloca 面板的主要可见文案已经接入翻译。
- About Veloca 的更新检查状态、更新操作、开源许可证弹窗和组件加载状态会跟随当前语言切换。
- Agent 对话框中的 tool / thinking 消息 item 已接入翻译，包括已知工具 action、thinking summary、工具状态 aria-label、Web Search turn 提示和 typing aria-label；工具返回的 detail 原文保持不变，避免改变工具结果含义。
- Appearance 语言切换会立即更新 `document.documentElement.lang`、`data-language`，并刷新已接入 `t()` 的界面文案。
- Toast 中与 Settings 相关的成功提示已经接入翻译。

## 后续要求

- 新增页面、弹窗、菜单、toast、aria-label 时，必须优先在 `i18n.ts` 中新增 key，再通过 `t()` 使用。
- 如果未来引入完整 i18n 框架，需要保持现有 key 语义稳定，避免破坏已接入的 Settings 文案。
- 后端错误信息仍保持英文为主，前端如需展示给用户，应在 UI 层映射为可翻译文案。
